# 小红书下拉词批量采集器（增强版）

基于开源项目 [chenyaoswu/xiaohongshu-search-keywords-collector](https://github.com/chenyaoswu/xiaohongshu-search-keywords-collector) 的 DOM 选择器重写增强。原版只能采集"当前已展开的下拉词"并复制到剪贴板；本版补齐了**批量种子词、递归长尾挖掘、CSV/TXT 导出、本地保存**。

## 功能
- **批量种子词**：一次粘贴多个核心词（每行一个），自动依次查询。
- **递归长尾**：可选 1~3 层。例如种子「虾仁」→ 下拉「虾仁 减脂」→ 再下钻「虾仁 减脂 做法」，指数级扩展长尾词。
- **自动去重**：全局按词去重，避免重复与死循环。
- **实时进度**：弹窗内显示当前词、层级、已采集数量、进度条。
- **一键导出**：CSV（含 种子词/层级/下拉词 三列，带 BOM 适配 Excel）或 TXT（全部唯一词）。
- **本地保存**：结果存于浏览器本地，关闭弹窗不丢失。
- **采集当前下拉**：手动在小红书搜索框输入并展开下拉后，一键抓取当前词。
- **上升趋势雷达**（v1.6）：每日定时快照种子词下拉词，自动差分出**新增词（上升趋势）/ 消失词（退潮）/ 持续词**，按"上升分"排序（新词 +50、位次分、连续天数），近 7 日词量柱图。新增词即抢写笔记的时机信号。

## 趋势雷达用法
1. 在「采集」页填好种子词（雷达复用这份种子词，固定 depth=1）。
2. 切到「趋势雷达」页，勾选「每日」并设时间（如 09:00），点「保存」。
3. 也可随时点「立即快照」手动跑一次。
4. 每天/每次快照后，页面显示新增词、上升分排序、近 7 日词量。点「导出趋势 CSV」可贴进腾讯文档搜索词看板。
> 定时依赖 `chrome.alarms`，需浏览器处于运行状态；快照最多保留 30 天。

## 安装（Chrome / Edge 均支持）
1. 下载本文件夹（或 `git clone` 原仓库后替换文件）。
2. 打开浏览器扩展管理页：
   - Chrome：`chrome://extensions`
   - Edge：`edge://extensions`
3. 右上角开启 **开发者模式**。
4. 点击 **加载已解压的扩展程序**，选择本文件夹（包含 `manifest.json` 的目录）。
5. 工具栏出现扩展图标即安装成功。

## 使用
1. 打开并登录小红书（`www.xiaohongshu.com`，建议停在 `/explore`）。
2. 点工具栏图标，在「种子词」框粘贴核心词（如 `虾仁` / `减脂餐` / `一人食` / `三文鱼`）。
3. 选递归层级与请求间隔（建议 ≥ 800ms，降低风控风险）。
4. 点 **开始采集**，弹窗实时滚动结果。
5. 结束后 **导出 CSV / TXT**，可直接贴进腾讯文档搜索词看板。

## 注意事项
- **登录态**：小红书下拉词在登录后更全，请先登录账号。
- **风控**：不要设过低间隔或超大批量，可能触发验证码；按需用 1~2 层。
- **DOM 变动**：小红书改版可能导致选择器失效。若采集为空，到 `content.js` 的 `SELECTORS` 补充新的下拉容器/条目选择器即可。
- **自动输入失效兜底**：若"模拟输入"未触发下拉（个别环境对合成事件敏感），可手动在搜索框输入并展开下拉，再用弹窗的「采集当前下拉」按钮抓取。

## 历史记录
- 每次**采集批次**和每次**快照**都会自动写入「历史」页（本地最多 100 条）。
- 「历史」页可按时间浏览、点开看详情、从云端同步、清空。
- 登录账号后，新记录会自动云同步；登出时仍可本地使用。

## 账号登录与云同步（基于 Supabase）
真正登录注册 + 跨设备同步，用免费 Supabase，无需自建服务器。

### 一次性配置
1. 到 https://supabase.com 注册并新建一个项目（免费档即可）。
2. 进 Dashboard → **SQL Editor**，粘贴运行仓库内 `supabase_schema.sql`（建表 + 行级安全策略）。
3. 进 **Project Settings → API**，复制 `Project URL` 和 `anon public` key。
4. 插件「账号」页填入 URL 与 anon key，点保存。
5. 在「账号」页用邮箱密码**注册 / 登录**。
6. 之后每次采集 / 快照自动写入云端；换设备登录后点「从云端同步」即可拉回历史。

> 安全说明：anon key 是公开密钥，配合表上的行级安全策略（RLS）使用——每个用户只能读写自己的记录。你的密码不经过插件作者，直接由 Supabase Auth 处理。

## 文件结构
```
manifest.json        插件配置（MV3）
content.js           页面内：模拟输入 + 网络拦截采集下拉词
inject.js            主世界：拦截小红书联想请求响应
background.js        后台：队列编排 + 递归 + 暂停 + 保活 + 历史 + 定时
radar.js             上升趋势雷达：快照/差分/上升分
supabase.js          账号登录 + 云同步（Supabase Auth + PostgREST）
popup.html/js        弹窗：采集 / 趋势 / 历史 / 账号 四个 Tab
supabase_schema.sql  云端建表 SQL
tests/               不变量测试（25 项）
```
